> 使用 CSS Grid Template Areas 創建直觀的 CMS 布局系統

當我們在設計複雜的 CMS 系統時，經常遇到版面配置的挑戰。傳統的 flexbox 或 float 方式雖然能實現功能，但在維護性和直觀性上總是差強人意。今天要介紹的 **Grid Template Areas** 提供了一個更優雅的解決方案。

## 為什麼選擇 Grid Template Areas？

<Callout type="info">
Grid Template Areas 讓你能用視覺化的方式定義布局，就像畫草圖一樣直觀！
</Callout>

<Steps>

<Step>
### 視覺化布局定義
直接在 CSS 中"畫出"你的版面配置，一目了然
</Step>

<Step>
### 具名區域管理
每個區域都有明確的名稱，避免複雜的選擇器
</Step>

<Step>
### 響應式設計友好
輕鬆調整不同螢幕尺寸的布局
</Step>

<Step>
### 高可維護性
修改布局時只需調整 template 定義，不需要動到 HTML 結構
</Step>

</Steps>

## 視覺化布局結構

典型的後台管理系統布局通常包含以下區域：

```
┌─────────────────────────────────────────┐
│                Header                   │
├───────────┬─────────────────────────────┤
│           │                             │
│  Sidebar  │         Main Content        │
│           │                             │
├───────────┼─────────────────────────────┤
│           │                             │
│  Sidebar  │         Footer              │
│           │                             │
└───────────┴─────────────────────────────┘
```

## Grid 尺寸設定詳解

<Callout type="warn">
理解不同的尺寸單位是創建完美布局的關鍵！
</Callout>

### 尺寸單位說明

<Tabs items={["Fixed Size", "Flexible Size", "Auto Size"]}>

<Tab value="Fixed Size">
```css
/* 固定尺寸 (px) - 適合需要精確控制的區域 */
.grid-container {
  grid-template-columns: 200px 1fr 200px;
  grid-template-rows: 60px 1fr 100px;
}
```

**適用場景：**
- Header 高度固定
- Sidebar 寬度固定
- Footer 高度固定
</Tab>

<Tab value="Flexible Size">
```css
/* 彈性尺寸 (fr) - 自動分配剩餘空間 */
.grid-container {
  grid-template-columns: 1fr 2fr 1fr;
  grid-template-rows: auto 1fr auto;
}
```

**適用場景：**
- Main content 區域自適應
- 響應式設計
- 不確定內容大小的區域
</Tab>

<Tab value="Auto Size">
```css
/* 自動尺寸 (auto) - 根據內容調整 */
.grid-container {
  grid-template-columns: auto 1fr auto;
  grid-template-rows: auto 1fr auto;
}
```

**適用場景：**
- 內容驅動的區域大小
- 動態導航選單
- 可變長度的文字內容
</Tab>

</Tabs>

```css
      "footer footer footer";

  }
}
```
</Step>

<Step>
### 平板版本 (Tablet)
```css
@media (max-width: 1023px) and (min-width: 768px) {
  .grid-container {
    grid-template-areas: 
      "header header"
      "sidebar-left main-content"
      "footer footer";
    grid-template-columns: 180px 1fr;
  }
  
  .sidebar-right { display: none; }
}
```

</Step>

## 結論

<Callout type="success">
Grid Template Areas 為複雜的 CMS 布局提供了完美解決方案
</Callout>

Grid Template Areas 的優勢總結：

1. **視覺化定義**：CSS 代碼即是布局草圖
2. **語義化命名**：區域名稱清晰易懂
3. **響應式友好**：輕鬆適配各種螢幕尺寸
4. **維護性佳**：結構調整不影響 HTML
5. **學習曲線平緩**：直觀易懂的語法

對於需要處理複雜版面配置的 CMS 系統來說，Grid Template Areas 絕對是值得投資學習的技術。它不僅能提升開發效率，更能讓代碼更具可讀性和可維護性。
